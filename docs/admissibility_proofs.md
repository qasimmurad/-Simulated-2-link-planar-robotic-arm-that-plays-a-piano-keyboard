# Heuristic Admissibility Proofs

This document formally argues the admissibility (and limitations) of
each heuristic used in the A* motion planner.

## Background

A heuristic h(n) is **admissible** if it never overestimates the true
minimum cost h*(n) to reach the goal from node n:

    h(n) ≤ h*(n)  for all n

A heuristic is **consistent** (monotone) if for every node n and
successor n' via action a:

    h(n) ≤ c(n, a, n') + h(n')

Consistency implies admissibility. A* with a consistent heuristic is
optimal when using a closed list. A* with an admissible-but-inconsistent
heuristic is optimal only if nodes are reopened when better paths are found.

The search cost in this project is total joint-angle travel:

    cost(path) = Σ |Δθ₁| + |Δθ₂|  across all transitions

## 1. joint_space_heuristic — Admissible ✓

**Definition:** For a current configuration (θ₁, θ₂) and a target note
position p, compute the exact minimum joint travel required to reach any
of the 6 candidate configurations for p:

    h(current, p) = min over configs c of (|c.θ₁ − θ₁| + |c.θ₂ − θ₂|)

**Proof of admissibility:**

Let S* be the optimal sequence of configurations from the current state
to the goal. At the very next step, the arm must move to one of the 6
candidate configurations for the next note. The heuristic returns the
minimum cost of doing so. Since the true cost must include at least this
one transition, h(n) ≤ h*(n). ∎

**Consistency:** This heuristic is consistent because it evaluates the
exact cost of the first step. It satisfies the triangle inequality by
construction.

## 2. h_euclidean_endeffector — Inadmissible on Wide State Space ✗

**Definition:** Straight-line Cartesian distance between the current
end-effector position and the target key position.

**Initial intuition (incorrect):** Cartesian distance is a lower bound
on arc-length, which should lower-bound joint travel.

**Why this fails on our wide state space:**

The wide state space includes perturbed configurations where the arm tip
is NOT at the target key position. Consider a perturbed configuration
generated by offsetting θ₁ by ±0.1 rad — the tip is displaced slightly
from the target. If the next note is the SAME key, the minimum cost
transition is zero (the search can expand to the identical perturbed
config in the next step's candidate set). However, h_euclidean sees the
tip displaced from the note position and returns a positive value.

**Counterexample (empirically verified):**

    Transition: E4 → E4 (same note, repeated)
    Current:    perturbed config with θ₁ offset +0.1 rad
    h_euclidean estimate: 0.0521 rad
    True minimum cost:    0.0000 rad (stay in same perturbed config)
    Violation:  h(n) > h*(n) ✗

This was discovered via automated diagnostic in test_search.py
(test_euclidean_heuristic_inadmissible_counterexample).

## 3. h_combined — Inadmissible on Wide State Space ✗

**Definition:** max(joint_space_heuristic, h_euclidean_endeffector)

**Why max() is the correct combiner when both components are admissible:**

If h₁(n) ≤ h*(n) and h₂(n) ≤ h*(n), then:
    max(h₁(n), h₂(n)) ≤ h*(n)  ∎

This is strictly more informed than either component alone.

**Why sum() would be wrong:**

h₁(n) + h₂(n) ≤ 2·h*(n), which does not imply h₁ + h₂ ≤ h*(n). The
sum can overestimate and would make A* inadmissible.

**Admissibility of h_combined:** Conditional. When both components are
admissible (exact IK configurations), h_combined is admissible. On
perturbed configurations in the wide state space, h_combined inherits
the inadmissibility of h_euclidean. In practice, h_combined finds
suboptimal plans on Ode to Joy (0.8% excess cost) for exactly this reason.

**Recommendation:** Use joint_space_heuristic (_wide_heuristic in the
search module) when optimality is required.
